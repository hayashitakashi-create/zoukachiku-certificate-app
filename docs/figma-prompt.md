## 💡 追加で伝えると良いこと

### デザイン途中で確認すべきポイント

```
【確認1: デザインシステム完成時】
カラーパレット、フォント、基本コンポーネントを確認させてください。
この段階で方向性を合わせておきたいです。

【確認2: Desktop版完成時】
レイアウト、余白、視覚的な優先順位が適切か確認します。

【確認3: レスポンシブ対応時】
Tablet、Mobile版の情報の見せ方を確認します。
```

### デザイナーからよく聞かれる質問への回答

```
Q: ロゴはありますか？
A: いいえ、システムタイトルのテキストのみです。

Q: 使用するアイコンライブラリは？
A: 現状は絵文字ですが、Heroicons、Lucide、またはMaterial Iconsを推奨します。

Q: 写真素材は必要ですか？
A: いいえ、データとUIのみのシステムです。

Q: ユーザープロフィール機能はありますか？
A: 現状はありません。将来的に追加予定です。

Q: ダークモードは必要ですか？
A: オプションです。まずはライトモードを優先してください。

Q: アニメーションはどの程度？
A: 控えめで品のあるマイクロインタラクション（ホバー、トランジション）程度で。

Q: アクセシビリティの基準は？
A: WCAG 2.1 AA レベルを目指しています。カラーコントラスト比 4.5:1 以上。
```

## 📸 スクリーンショット提供のヒント

### 撮影すべき画面
1. ダッシュボード（データあり）
2. ダッシュボード（空状態）
3. 証明書詳細ページ（参考）
4. 統計カードのクローズアップ
5. テーブルのクローズアップ

### 撮影方法
```bash
# ブラウザで http://localhost:3000 を開く
# デベロッパーツールで画面サイズを 1440x900 に設定
# スクリーンショットを撮影
```

## 🎯 段階的な依頼方法（推奨）

### アプローチ1: 段階的デザイン（推奨）

```
【Phase 1】
まずはデスクトップ版のデザインシステムと基本レイアウトをお願いします。
→ レビュー → 修正

【Phase 2】
レスポンシブ対応（Tablet、Mobile）をお願いします。
→ レビュー → 修正

【Phase 3】
インタラクションプロトタイプを作成してください。
→ レビュー → 最終調整
```

### アプローチ2: 一括依頼

```
すべて（デザインシステム + Desktop + Tablet + Mobile + プロトタイプ）を一度にお願いします。
完成後にまとめてレビューします。
```

**推奨**: アプローチ1（段階的）
- デザインの方向性を早めにすり合わせできる
- 手戻りが少ない
- デザイナーも作業しやすい
